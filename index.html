Anti-patterns détectés

Les problèmes que nous
identifions pour vous

CRITIQUE
Cartesian Product
Join sans condition → complexité O(n×m). Peut faire crasher votre cluster.
ATTENTION
SortMergeJoin évitable
2 shuffles + 2 sorts inutiles si une table est broadcastable.
ATTENTION
Data Skew
Tâches déséquilibrées (ratio >5x) → une partition ralentit tout le job.
ATTENTION
Spill mémoire
Données écrites sur disque → ralentissement drastique des shuffles.
INFO
200 partitions par défaut
Valeur par défaut rarement optimale. Trop élevé ou trop faible selon le volume.
ATTENTION
Pas de Predicate Pushdown
Toutes les données lues avant filtrage → I/O inutiles sur le stockage.
